More Details

Prototype - Endless runner(Solo Unity)
More Details

Created a small project similar to an endless runner. The objects are activated and deactivated every time the chunk is brought to the initial position. The player has to avoid the red cubes, collect coins and optionally can heal with the green spheres. The game becomes more difficult over time as the speed of the chunks gets faster and faster. The game has a saving system which allows for upgrades for the player like more life, a one use shield and others.

Tools - Saving and Upgrade System(Unity)
More Details

A small tool for creating saving and shop systems. The artefact has an usable player, randomly spawning coins and a shop. The coins are spawned by a manager in a defined, changeable volume and limited in number. The coins can be used in the shop for different upgrades like additional jumps.

Prototype - Procedural Texture(Unreal)
More Details

A small survival/escape game where you can hunt animals and interact with shrines in order to leave the area. Main attraction is the terrain texture thar procedurally applies specific or mixture of sub-textures in order to give a smooth transition between the the higher and lower areas on the map.

Prototype - Procedural Objects Placement(Unity)
More Details

A prototype for a procedural object placer. It takes a plane which can be resized. Depending on the available area, the manager will set a maximum number of objects it can spawn. This can be tweaked for lower values. The objects will afterwards be placed in such a way that none will overlap. The maximum amount of objects at the time in the scene allowed by the editor manager is 2000 spawned units, though, through code you can spawn up to 11000 units with little to now delay. The placer can spawn AIs, interactable objects and obstalces.
